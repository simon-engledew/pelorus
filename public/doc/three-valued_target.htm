Three-valued target

At the advanced level, a number or date factor can have a three-valued target, rather than the single-valued target of the basic level. The three values are:

•target

The value that the stakeholders in the goal are aiming to achieve. This value must be prefixed by a direction indicator (> or <) and—in the case of a number factor—can be suffixed by a units indicator.

•report

A value between the target and fail values at which all impacted stakeholders must be informed.

•fail

The worst value that would be acceptable. If the actual value of the factor is worse than this fail value, then the factor—and hence the entire goal—has not been achieved.

Where three-valued targets are specified, the predicted/actual values must also be three-valued. These values are:

•best

The best value that the stakeholders currently believe they may be able to achieve.

•likely

The value that the stakeholders currently believe they are likely to achieve.

•worst

The worst value that the stakeholders currently believe they will achieve.

Where the figure is actual rather than predicted, the three values are of course all the same.

The traffic-light status of the factor is then calculated using the following rules in order:

if worst is worse than fail, the status is red

if likely is worse than report, the status is amber

otherwise the status is green.

Note that neither the target nor the best values are used in calculating the status. If stakeholders want to be informed about the predicted or actual value being worse than the target, they should set the report value to be the same as the target value.
