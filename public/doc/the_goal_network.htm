The goal network

A map has a unique name. It also has a description, which is simply a piece of text that describes its purpose.

A map takes the form of a network. Specifically it consists of a set of goals that are inter-connected by links. In the example map shown below, the boxes represent the goals while the arrows represent the links.

A goal is a steady state, not a dynamic activity—it’s not a thing we’re aiming to do but rather a place we’re aiming to be.

Each goal has a name that distinguishes it from other goals and indicates the steady state that the goal represents . So, for example, one goal in the example map is named Work Reallocated, which suggests a steady state. To have named the goal Reallocate Work, which suggests a dynamic activity, would have been misleading and wrong.

Links are directed—they have an arrowhead at only one end. Link cycles are forbidden. So the map takes the form of an acyclic directed graph.

A link from goal A to goal B—such as that from Customer Enquiry Process Redesigned to Work Reallocated—indicates that A supports B. This means simply that the stakeholders in goal B believe that achievement of goal A will facilitate their own achievement of goal B. This is deliberately a weak statement. There is no concept of the supporting goals being sufficient to ensure achievement of the supported goal, nor even of their being necessary—conceivably it would still be possible to achieve goal B even if goal A were missed, though presumably it would be more difficult.

Any goal may have any number of sub-goals. This is basically a grouping mechanism—it provides the option of defining the internals of a goal through hierarchical decomposition rather than as a single ‘flat’ structure. Sub-goals may themselves have links, such as that from the Pilot System Operational goal in the example map below.
