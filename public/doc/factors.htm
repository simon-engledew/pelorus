Factors

A goal can have any number of factors. Each factor has a name, a description and a target value.

Collectively, the factors define the goal. A goal is achieved by simultaneously achieving the target values for all its factors and achieving all its sub-goals.

A factor can be any one of three types—a number, a date, or Yes/No (Boolean). Number values can either be integers (e.g. 127) or decimal (e.g. 49.63). Dates are formatted as either dd/mm/yy or dd/mm/yyyy. Yes/No values are formatted as either the single character Y or the single character N.

For some factors (such as Profit) a higher value is better, while for other factors (such as Cost) a lower value is better. For factors of type number and date, the target value is therefore prefixed by a mandatory direction indicator, which is either a > or a <. The > indicator shows that higher values are better, while the < indicator shows that lower values are better.

For factors of type Yes/No, the target value is prefixed by =. So the only possible targets for Yes/No factors are =Y and =N.

Optionally, number factors can be suffixed by a units indicator—a short sequence of non-numeric characters that indicates the nature of the number. For example, the target for one factor could be specified as <500 £K while that for another factor could be >92%.

At some point during the life of the map—in the Delivery phase, if not before—the Goal Manager will provide predicted or actual values for factors. A predicted value looks to the future: it is the value that, on the basis of all available information, the stakeholders in the goal now believe they will be able to achieve. An actual value is the value that has actually been achieved once efforts to reach the goal have ceased.

The relationship between the target value and the predicted/actual value generates a traffic-light status for the factor.

For a number or date factor, if the predicted/actual value is ‘worse than’ the target value (which of course depends on the direction indicator) the traffic-light status is red, otherwise it is green. Although the direction indicator symbols are > and <, a predicted/actual value that is equal to the target value is regarded as green. (So the true meaning of the > and < direction indicators is actually ≥ and ≤ respectively.)

For a Y/N factor, the predicted/actual value must be one of Y, N or ?. The traffic-light status is then:

•green if the predicted/actual value matches the target value (i.e. both are Y or both are N)

•red if the predicted/actual value is the opposite of the target value (i.e. one is Y and one is N)

•amber if the predicted/actual value is ?.

In all cases, if no predicted/actual value has been provided, the status is green.
